Cognitive Design principles

  • Attention Principles
  • "Don't make me think" user design in which the user can get what they want without using critical thought.
  • Distractions should be reduced so the user needs to use less focus.
  • Perception Principles
  • Information should be presented clearly and precisely.
  • Things should not trick the user into being double meanings or similarities should be reduced.
  • Memory Principles
  • Working Memory Vs Long Term Memory.
  • Working Memory should have creation of gui and visual or auditory interfaces.
  • Long term memory should be used to have standards set be implemented in your design. For example menu in top right and logos in top left.
  • Other Principles
  • Mental models should be accounted for to adjust to expectations from a potential user.
  • Feedback from users, and testing in the design phase.

    • Key Principles

  • Visibility Principles
  • Elements should be visible and not have conflicting patterns and colors.
  • Consistency Principles
  • Elements should be the same throughout with things like menu buttons not moving from one page to the next.
  • Feedback Principles
  • Elements that users find unpleasant or lacking should be reported to the designer so that action can be taken.
  • Constraint Principles
  • Options for the user should be made to provide appropriate responses and stop errors.
  • Affordance Principles
  • Elements should provide distinction to the user if they hold more visual weight.
  • Accessibility Principles
  • The user should have the ability to interact with your design if there are inabilities on behalf of the user.

    • Gender Principles

  • Gender design involves designing systems and interfaces that are inclusive of a persons gender expression and allow for less restrictive user gender selection.
  • Additionally gender and user gender input should not affect the abilities site to provide a service for the users needs.

    • Ethical Principles

  • Designers can focus certain standards and norms for genders like potential for having ethical user targeted advertisements.
  • Designers should always provide options for the user to allow for "opt-out by default" unnecessary user data collection.

    • Emotional Principles

  • Emotional design should aim to invoke good emotions.
  • Designs should also aim to remove frustration that may come to the user.

    • Accessibility Principles

  • Accessibility should always be considered. Accessibility can come in many forms both physical and mental.
  • Accessibility should help users with different tools available. Screen readers, auditory prompts, subtitles.
  • Universal design should help all users regardless of ability.

    • Error Principles

  • Errors are not the users fault
  • Humans will always make errors, but it is the designers job to make sure that the errors are minimized to maximum degree.
  • If error is to happen then the designer may want to make sure that recovery after is quick and reduces impact on the system.

    • Complexity Principles

  • Be as easy as email!
  • The less complexity you have the more you can be accessible to users with intellectual disabilities.

    • Tesler's Principle

  • Simple designs are important however tesler's law says a minimum amount of complexity will always exist.

    • Interface Principles

  • Consistency should be present in the design.
  • The design should incorporate universal mental models to help ease the user and make them not think.

    • Cultural Principles

  • Cultural diversity should be accounted for as different cultures will have different standards and norms.

    • Language Principles

  • Different languages are spoken for around the world and many people may not be able to speak the same language as you.
  • A user may read differently not just in language but in orientation.

    • Conceptual Principles

  • Modular based design can help designers organize in the case of reiterations or updates.
  • Clarity can be given if the user groups things into organized containers that can be lifted easily from the system.

    • Human Center Design Principles

  • HCD involves making sure that the needs of the users are met as well as improving the system.

    • Diversity Based Design

  • The design should be made with many people in mind by many designers so that the needs are properly met.

    • Context based Design

  • Designs should ensure that the system fits in the users circle, for example a phone app shouldn't be made to perform the duty of an action that can be done more simply by a user.
