Module 7 – Chapters

  • Interfaces – the WIMP model

    • Windows, Icons, Menus, and Pointers

    Windows is the rectangular box and frames, icons are the small graphical representations tat represent the file, menus are hierarchical lists of options, and pointers are mouse and trackpad interactions.

  • Isomorphic icons

    • Refer to graphical symbols that are designed to have consistent visual appearences. A home icon being the main icon for the main menu is an example.

  • Typography considerations
  • Website breadcrumb navigation

    • Breadcrumb navigation is a type of website navigation that provides users with a hierarchical trail of links that indicate the user's location within the website's structure.

  • VR, AR
  • Voice-to-text
  • Context-based advertising

    • Context-based advertising is a form of advertising the targets ads to a user based onthe content of the website. Targeted ads allow for more relevant advertising to a user.

  • How to configure pages for scanning

    • Use simple language.

    Break down complex tasks into simple steps.

    Provide visual aids.

  • Visual hierarchies – vertical vs horizontal

    • Visual hierarchy refers to arrangement and organization of visual elements.

  • Optimizing user choices

    • Limit the number of choices the user has.

    Group choices into categories.

    Use clear and concise language.

    Provide guidance and feedback.

  • Home pages
  • Navigation

Module 8 – Chapters

  • Prototypes-Low vs high fidelity, issues

    • Low fidelity prototypes are proof of concepts and are low quality versions, so they have difficulty expressing details and functionality.

    High fidelity prototypes usually come after and express more detail and functions, but they are more expensive and timely to create.

  • The "wizard of oz" model

    • Prototyping technique used in the design of interactive systems. It involves simulating the functionality without building it. It also allows for users to test the product multiple times.

  • Storyboards

    • Storyboards are visual representations of a narrative. They allow for a made up scenario and user to express the need of a system that will be built.

  • Conceptional models & metaphors
  • The Doherty Threshold

    • The threshold (400 ms) that users will perceive as an instantaneous response. Fast response is important as breaks longer than a second are perceived as a delay.

  • Wireframes

    • Wireframes are visual representations of a system. These are made to signify how one event would lead to another without being bogged down by details.

  • Gamification

    • Process of using game elements to make the user motivated to use the site. These could be tasks or fake rewards that engage the user.

  • Design Thinking
  • Arduinos and Microbits
  • SDK's

    • (Software Development Kit) These are tools made for others to develop applications. SDK's are intended to save developers time by providing them tools and allow for easier tools.

Module 9 – Chapters

  • Use cases: actors, goals, system

    • Use cases are ways to model interactions between users of the system.

    Actors are any outside force acting upon the system, this could be a user, another system, or a combined entity.

    Goals are objectives that the actor(s) are trying to achieve with the system.

    System is the software or platform that actors use to acheive their goals

  • Databases-Reality, data, metadata

    • Databases store information, Reality refers to real world entities or objets represented. Data refers to information stored within the database. Metadata is data about data, this can include information like data types.

  • Four step process of database design
  • Database lookup tables - what & why

    • A reference table lists values and codes. The purpose is to keep the data consistent, that only valid entries are within the table, and improve maintenance and performance.

  • Key database design best practices
  • Database anomalies
  • Data warehouse, lake, mining

    • Data warehouse is a repository of data used for business intelligence to integrate data. Warehouses "prepackage" the data and make it easy to use.

    Data lakes are similar except they are made to scale easily and the data is not readily available for use.

    Mining is the process of studying large datasets and extracting information that matters.

  • ER diagrams

    • Entity Relationship Diagrams

    Entities are people place or object, Attributes are characteristics of the entity like names and ID's, and relationships are made between the entities to show correlation between the two.

  • Data visualization
  • Displaying 'centrality'
  • Dashboards

    • Dashboards are visual representations of data, used to monitor and analyze performance indicators, metrics, and other data points.

    Dashboards should use Key Performance Indicators (KPIs), metrics, visualizations, filters, and alerts.

  • Human-centered design
  • Ethical concerns: FATE

    • Fairness, Accountability, Transparency, Ethics

    These principles can be exacerbated by AI and ML as systems may perpetuate existing biases or discriminate against certain groups. As designers, we are not responsible for actions. Transparency should allow for users to see how these AI models make decisions. Ethics dictate these systems should only look to improve life quality.

  • Data formatted in XML

    • eXtensible Markup Language, is a markup language which use elements, tags, attributes, namespaces, and structured in certain ways.

  • JS data types, arrays, objects

    • JS uses things like Strings, Ints, and Booleans as data types.

  • JSON

    • JavaScript Object Notation. JSON files allow for information exchange that can be deciphered by multiple programming languages.
